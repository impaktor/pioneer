SystemBody

Class representing a system body.

SystemBody differs from Body in that it holds the properties that are used to generate the physics <body> that is created when the player enters a system.  It exists outside of the current space.  That is, scripts can use a SystemBody to discover information about a body that exists in another system.

lifeReturns the measure of life present on the body 0.0 = dead, 1.0 = teeming (~= pandora)
hasRingsReturns true if the body has a ring or rings of debris or ice in orbit around it
hasAtmosphereReturns true if an atmosphere is present, false if not
isScoopableReturns true if the system body can be scoopable, false if not

seed

The random seed used to generate this SystemBody.  This is guaranteed to be the same for this body across runs of the same build of the game, and should be used to seed a Rand object when you want to ensure the same random numbers come out each time.

This value is the same is the one available via Body.seed once you enter this system.
